# 📌 SEO 최적화 글쓰기 지침 (Master Guidelines)

## 【SEO 최우선 체크리스트】
*   **메인 키워드:** 사용자 입력 주제.
*   **제목(h1):** 30자 이내, 메인 키워드 포함, 클릭 유도 (예: "사과 보관법 3개월 신선하게").
*   **소제목(h2):** 7~8개 중 2~3개만 메인 키워드 포함. 나머지는 LSI/세부 키워드.
*   **키워드 밀도:** 첫 문단 2회, 본문 300자마다 1회, 마지막 문단 1회, FAQ 4/8개 포함.
*   **LSI 키워드:** 연관 검색어 자연스럽게 녹이기.

## 【구조 및 분량】
*   **총 분량:** 주제의 깊이에 따라 **4000~6000자** 사이로 유연하게 조절. 무리한 늘리기보다는 알찬 정보 위주.
*   **구성:** 
    1.  **H1 (제목)**
    2.  **인트로:** 2-3문장. "제공하겠습니다" 식의 선언형 금지. **"함께 알아봐요", "살펴볼게요"** 등 청유형/공감형으로 작성.
    3.  **스타일(CSS):** TOC, Table 등 디자인 정의 (문서 내 1회).
    4.  **목차(TOC):** 5~8개 항목 + FAQ. 연한 하늘색 박스 스타일.
    5.  **본문(Body):** 5~8개 섹션.
        *   정보성: 도입 -> 블릿포인트(핵심) -> 보충설명.
        *   가독성: 3문장마다 줄바꿈.
        *   **표(Table):** 핵심 정보를 담은 주력 섹션 위주로 **유연하게 2~3개 배치**. (스니펫 노출 및 가독성 최적화)
    6.  **관련 정보 더 보기:** 외부 링크 영역.
    7.  **FAQ:** 질문 8개 / 답변 8개 필수.
    8.  **면책 조항:** FAQ 뒤에 배치.
    9.  **태그:** #없이 콤마로 구분.

## 【스타일 및 제약사항】
*   **어조:** 부드러운 구어체 ("해요", "합니다" 혼용). 전문 용어 풀이.
*   **HTML:** `data-ke-size` 및 인라인 `style` 속성 사용 금지. 모든 스타일링은 CSS 클래스나 `<style>` 블록으로 처리한다.
*   **TOC 스타일:** 목차 링크에 `hover` 시 링크 텍스트의 색상 변경 및 밑줄, 그리고 `<li>` 항목 전체의 배경색 변화 등을 통해 시각적으로 명확하고 동적인 느낌을 주되, JavaScript 없이 CSS만으로 구현. SEO 친화적인 범위 내에서 최적의 사용자 경험을 제공한다.
*   **금지어:** "결론적으로", "마지막으로", "하지만", "또한", "놀라운", "충격적인", "대박".
*   **형식:** **전체 HTML을 하나의 `copycode` 코드 블록으로 출력.**
*   **텍스트:** `**` (볼드) 문법 금지. `<strong>` 태그 사용.
*   **이미지:** 텍스트로만 출력 (이미지 출력 오류 방지).
*   **최신 정보 표현 지침:** "최신 정보에 따르면", "2026년 최신"과 같은 표현을 과도하게 반복하거나 글의 흐름을 방해할 정도로 강조하지 않는다. 연도나 특정 시점이 정보의 핵심적인 부분인 경우에만 명확하게 언급한다.
*   **최신 정보 표현 지침:** "최신 정보에 따르면", "2026년 최신"과 같은 표현을 과도하게 반복하거나 글의 흐름을 방해할 정도로 강조하지 않는다. 연도나 특정 시점이 정보의 핵심적인 부분인 경우에만 명확하게 언급한다.

## 【주제 집중 및 내용 확장】
*   **주제 집중:** 메인 키워드와 직접적인 LSI 키워드에만 집중하여 글 작성. 연관성이 떨어지는 정보는 최대한 배제하여 **미래의 다른 포스팅과 내용 중복이 발생하지 않도록 한다.**
*   **내용 확장:** 6000자 이상을 채우기 위해 불필요한 반복이 아닌, **해당 주제의 깊이와 폭을 확장하는 방향**으로 내용을 구성. 독자가 정말 궁금해할 만한 심화 정보를 제공.

## 【작성 프로세스】
1.  **검색 (Search):** 2026년 최신 데이터 구글 검색 필수.
2.  **설계:** H1, H2, 목차 구성.
3.  **작성:** 섹션별로 내용을 풍부하게 작성 (Chaining).
4.  **출력:** 최종 HTML 코드로 변환하여 출력.
