# 프로젝트 블루프린트

## 1. 프로젝트 개요
이 프로젝트는 웹 기반의 간단한 게임들을 개발하는 것을 목표로 합니다. 현재는 "Cosmic Fortune"이라는 이름의 "오늘의 운세" 게임이 구현되어 있으며, 사용자의 피드백을 반영하여 지속적으로 개선하고 있습니다.

## 2. 구현된 기능 및 디자인

### 2.1. 오늘의 운세 게임 (Cosmic Fortune)
사용자가 하루에 한 번 오늘의 운세 점수를 확인하고, 그 점수에 따라 격려의 메시지와 시각적인 피드백을 받는 웹 컴포넌트 기반 게임입니다.

*   **핵심 기술:** 순수 JavaScript, CSS, HTML을 사용하며, 재사용성을 위해 웹 컴포넌트로 모듈화되어 있습니다.
*   **주요 기능:** 금전운/인간관계운 제공, 하루 한 번 제한, 상태 유지, 동적 점수 애니메이션 등.
*   **"Bold" 디자인 시스템 적용:** 과감한 타이포그래피, 생동감 있는 색상/배경, 입체적 카드 디자인, 직관적 아이콘, 반응형 레이아웃, 라이트/다크 테마.

### 2.2. 최근 개선 사항
*   **버튼 스타일 버그 수정:** Shadow DOM 내부 스타일 정의로 해결.
*   **마크다운 렌더링 오류 수정:** `<strong>` 태그로 교체하여 해결.
*   **배포 캐시 문제 해결 시도:** CSS/JS 파일 버전 관리(`style.v2.css`, `main.v2.js`)로 캐시 파괴 시도.

## 3. 현재 직면한 핵심 문제

*   **GitHub Pages 배포 실패 (`523` 에러):** 최신 코드가 `git push`로 성공적으로 반영되었음에도 불구하고, 실제 웹사이트에는 적용되지 않는 문제가 지속되고 있습니다. 사용자가 직접 확인한 결과, 배포 과정에서 **Cloudflare `523` 에러(Origin Unreachable)**가 발생했으며, 이는 우리 코드가 아닌 GitHub Pages 서버 인프라의 문제임을 시사합니다.
*   **마지막 시도:** 이 문제를 해결하기 위해, `blueprint.md` 파일을 수정하여 강제로 재배포를 유도하는 '노크' 작전을 실행하고 `push`를 완료했습니다.

## 4. 내일을 위한 작업 계획

푹 쉬고 새로운 마음으로 다음 단계를 진행합니다.

### 4.1. 1단계: 마지막 배포 결과 확인
1.  **GitHub 저장소의 [Actions] 탭 확인:** 가장 최근에 실행된 배포 워크플로우의 결과를 확인합니다.
    *   **[성공] (초록색 체크):** 배포가 성공했다면, 문제는 해결되었을 가능성이 높습니다. 브라우저의 **강력 새로고침 (`Ctrl+Shift+R` 또는 `Cmd+Shift+R`)** 또는 **시크릿 모드**로 접속하여 최종 확인합니다.
    *   **[실패] (빨간색 X):** 여전히 실패했다면, 해당 워크플로우의 로그를 클릭하여 `523` 에러가 계속 발생하는지, 혹은 다른 에러 메시지가 있는지 확인합니다.

### 4.2. 2단계: 차선책 (만약 문제가 지속된다면)

GitHub Pages가 계속 불안정하다면, 더 이상 같은 방법을 고수할 수 없습니다. "깃(Git)으로 해야 한다"는 당신의 의지를 존중하여, Git 중심의 워크플로우를 유지하면서 안정성을 확보할 수 있는 다음 전략을 실행합니다.

*   **전략:** **GitHub Actions를 통해 Firebase Hosting으로 자동 배포**
*   **설명:** `git push`를 하면, GitHub Actions가 자동으로 코드를 가져와서 GitHub Pages가 아닌, 더 안정적인 **Firebase Hosting**으로 배포하도록 파이프라인을 구축합니다. 이렇게 하면 우리의 개발 방식(Git)은 그대로 유지하면서, 불안정한 배포 문제를 근본적으로 해결할 수 있습니다.
*   **실행 계획:**
    1.  Firebase 프로젝트 설정 및 Hosting 초기화.
    2.  GitHub Actions에서 Firebase에 접근할 수 있도록 인증 키 설정.
    3.  `.github/workflows/deploy.yml` 워크플로우 파일을 작성하여 `push` 이벤트가 발생할 때마다 자동으로 Firebase에 배포하는 로직을 구현합니다.

### 4.3. 3단계: 모바일 반응형 및 UI 개선 (완료)
사용자의 피드백을 반영하여 모바일 환경에서의 가독성, 레이아웃 안정성, 화면 맞춤 문제를 해결했습니다.

*   **문제점:**
    *   화면이 옆으로 흔들리는 불안정한 느낌 (Horizontal Scrolling issue).
    *   내용이 많지 않은데 스크롤을 해야 하거나, 한 화면에 딱 고정된 느낌이 들지 않음.
*   **해결책:**
    *   **가로 스크롤 완전 차단:** `body`와 `html`의 `width`를 제어하고 `overflow-x: hidden`을 엄격하게 적용.
    *   **배경 고정:** 배경의 빛나는 원(Orb)들을 `position: absolute`에서 `position: fixed`로 변경하여 스크롤 시에도 배경이 따라오지 않고, 레이아웃에 영향을 주지 않도록 수정.
    *   **한 화면 맞춤(One-screen fit) 시도:** 모바일에서 폰트 크기, 패딩, 마진을 대폭 축소하여 불필요한 스크롤 없이 한 눈에 정보가 들어오도록 최적화.
    *   **박스 모델 안정화:** 모든 요소에 `box-sizing: border-box`를 적용하여 패딩이 너비를 초과하지 않도록 수정.

### 4.4. 4단계: 시각 효과 복구 및 다국어 지원 (완료)
사용자의 요청에 따라 사라진 조명 효과를 복구하고, 한국어/영어 전환 기능을 추가했습니다.

*   **조명(Orb) 효과 복구:**
    *   `z-index: -1` 설정으로 인해 배경색 뒤로 숨겨졌던 조명 효과를 `z-index: 0`으로 수정하여 다시 보이게 복구. (콘텐츠는 `z-index: 1`로 유지하여 가독성 확보).
*   **다국어(Bilingual) 지원 구현:**
    *   **KO/EN 전환 버튼:** 메인 카드 상단에 언어 전환 버튼 추가.
    *   **번역 데이터 구축:** UI 텍스트(제목, 버튼 등), 운세 메시지, 명언에 대한 영문/국문 데이터를 `main.v2.js` 내에 구조화.
    *   **동적 언어 변경:** 버튼 클릭 시 페이지 새로고침 없이 즉시 텍스트가 해당 언어로 변경되도록 구현.
    *   **설정 유지:** `localStorage`를 사용하여 브라우저를 닫았다 열어도 선택한 언어가 유지되도록 설정.

### 4.5. 5단계: Formspree 제휴 문의 폼 추가 (완료)
사용자의 요청에 따라 Formspree를 활용한 간단한 제휴 문의 폼을 페이지 하단에 추가했습니다.

*   **목적:** 외부 연동 학습 및 사이트의 확장성 시연.
*   **구현:**
    *   `index.html`에 "Cosmic Collaboration Inquiry" 섹션과 Formspree 양식 추가.
    *   `style.v2.css`에 기존 테마에 어울리면서 반응형을 고려한 폼 스타일링 추가.
    *   `main.v2.js`에 폼의 제목, 입력 필드 라벨, 버튼 텍스트, 안내 메시지 등에 대한 다국어(KO/EN) 지원 추가.
*   **활성화 방법:** 폼의 `action` 속성에 있는 `YOUR_FORMSPREE_ID`를 Formspree.io에서 발급받은 실제 ID로 교체해야 합니다.
